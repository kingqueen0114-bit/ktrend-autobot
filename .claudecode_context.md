# Claude Code Context - K-Trend AutoBot

このファイルは、AI（Claude Code、Cursor等）がプロジェクトを扱う際の重要なコンテキストとルールを定義しています。

## 🔐 セキュリティ管理方針

### ❌ 禁止事項

**1Password CLIの使用禁止**
- このプロジェクトでは、1Password CLI (`op` コマンド) の使用を **一切禁止** します
- `op run`, `op inject`, `op read` などのコマンドは使用しないでください
- 1Password関連の設定やドキュメントの追加・提案も不要です

### ✅ 推奨事項

**ローカル開発環境**
- `.env` または `.env.yaml` ファイルを使用して秘密鍵を管理します
- これらのファイルは `.gitignore` に含まれ、リポジトリにコミットされません
- 開発者は手動で `.env.yaml` を作成・管理します

**本番環境 (Google Cloud)**
- **Google Cloud Secret Manager** を使用して秘密鍵を管理します
- Cloud Functions デプロイ時は `--env-vars-file .env.yaml` でローカルの環境変数をデプロイするか、Secret Managerから直接参照します
- Secret Manager使用例:
  ```bash
  gcloud functions deploy FUNCTION_NAME \
    --set-secrets="GEMINI_API_KEY=GEMINI_API_KEY:latest"
  ```

## 🛠️ 環境変数の取り扱い

### 必須環境変数

このプロジェクトで使用される環境変数：
- `GEMINI_API_KEY`: Google Gemini API キー
- `GOOGLE_CUSTOM_SEARCH_API_KEY`: Google Custom Search API キー
- `GOOGLE_CSE_ID`: Google Custom Search Engine ID
- `LINE_CHANNEL_ACCESS_TOKEN`: LINE Bot チャネルアクセストークン
- `LINE_CHANNEL_SECRET`: LINE Bot チャネルシークレット
- `LINE_USER_ID`: LINE 通知先ユーザーID
- `GCP_PROJECT_ID`: Google Cloud プロジェクトID
- `WORDPRESS_URL`: WordPress サイトのURL
- `WORDPRESS_USER`: WordPress 管理者ユーザー名
- `WORDPRESS_APP_PASSWORD`: WordPress Application Password

### ローカル開発時の設定手順

1. プロジェクトルートに `.env.yaml` ファイルを作成
2. 上記の環境変数を YAML 形式で記述:
   ```yaml
   GEMINI_API_KEY: "your-api-key"
   GOOGLE_CUSTOM_SEARCH_API_KEY: "your-api-key"
   # ... その他の環境変数
   ```
3. `.env.yaml` が `.gitignore` に含まれていることを確認

## 🤖 AI エージェント向けの注意事項

1. **機密情報の扱い**
   - 環境変数の値をログ出力やコンソールに表示しないこと
   - `.env.yaml` の内容を読み取った場合でも、その値を直接ユーザーに表示しないこと

2. **コード生成時の原則**
   - 環境変数は `os.getenv()` または `os.environ` 経由で取得すること
   - ハードコーディングされたAPIキーやトークンを含むコードを生成しないこと

3. **デプロイメント**
   - Cloud Functions のデプロイには `--env-vars-file .env.yaml` を使用
   - または、Secret Manager からの参照を推奨

4. **既存コードの尊重**
   - 既存のコードスタイル、ディレクトリ構成を維持すること
   - 不要な抽象化やリファクタリングを避けること

## ⚠️ 過去のトラブルと教訓

### 🔴 requirements.txt の google-generativeai パッケージ
- **絶対にやってはいけないこと:** `google-generativeai` を `google-genai` に変更する
- コード内では `import google.generativeai as genai` を使用しており、パッケージ名を `google-genai` にするとこのimportが解決せず、**Gemini AI機能が全面停止**する
- 将来 `google-genai` に移行するには、全importを `from google import genai` に書き換える作業が必要

### 🔴 Geminiモデル名
- `gemini-2.0-flash` を使用中（`gemini-1.5-flash` は廃止済み → NotFoundエラー）
- 変更箇所: `src/fetch_trends.py`, `src/content_generator.py`

### 🟡 f-string内のHTML/JavaScript
- `cloud_entry.py` の `view_draft` 関数はf-stringでHTMLを生成している
- `{}` → `{{}}` にエスケープ必須
- JS template literal `${var}` は使用不可 → 文字列連結で代替
- HTML属性値内の `>` は `&gt;` にエスケープ必須（HTMLパーサーが壊れる実例あり）

### 🟡 Google Custom Search API
- 403 Forbiddenが断続的に発生する既知問題
- `fetch_trends.py` の `_search_with_gemini` でフォールバック実装済み

### 🟡 LINE SDK
- SDK v1（`linebot`）を使用。v3（`linebot.v3`）ではない
- FlexMessage + message action でカテゴリ選択を実装（PostbackAction不使用）

## 📐 アーキテクチャ概要

```
main.py                    # Cloud Functions エントリポイント（ルーティング）
cloud_entry.py             # 全ビジネスロジック（LINE webhook, view_draft, 通知等）
src/
  fetch_trends.py          # トレンド取得（Google Custom Search + Geminiフォールバック）
  content_generator.py     # AI記事生成（CMS記事, SNS投稿, X投稿案）
  storage_manager.py       # Firestore + WordPress + GCS 連携
  notifier.py              # LINE通知
setup_richmenu.py          # リッチメニュー設定スクリプト
deploy.sh                  # デプロイスクリプト
```

### 主要エンドポイント
- `/webhook` — LINE Bot webhook
- `/draft/{id}` — 記事編集画面
- `/approve`, `/reject` — 記事承認・却下
