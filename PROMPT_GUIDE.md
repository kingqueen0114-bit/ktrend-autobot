# 記事生成プロンプト編集ガイド

K-Trend AutoBot の記事生成に使用されるプロンプト（AIへの指示書）は、プログラムコードから切り離され、以下のファイルで管理されています。

**ファイルパス:** `src/content_prompts.py`

このファイルを編集することで、記事の文体、構成、ルールなどを自由に変更できます。

---

## 編集方法

`src/content_prompts.py` はPythonファイルですが、中身はテキストの設定値（定数）が並んでいるだけですので、テキストエディタで簡単に編集できます。

### 主な設定項目

| 項目名 | 説明 |
|---|---|
| **EDITOR_ROLE** | AIの役割定義（ペルソナ）。「K-Trend Timesの編集者」などの設定。 |
| **BASIC_RULES** | 基本原則。「一次情報優先」「です・ます調」などのルール。 |
| **INPUT_TYPE_RULES** | 入力ソース別の処理ルール（Instagram画像向け、プレスリリース向けなど）。 |
| **BRAND_RULES** | ブランド名の表記揺れ統一や、特定チェーン店の記事ルール。 |
| **X_POST_RULES** | X（旧Twitter）投稿の文体、ハッシュタグのルール。 |
| **OUTPUT_FORMAT_ARTICLE** | CMS入稿用記事の出力フォーマット（JSON形式）。 |
| **OUTPUT_FORMAT_SNS** | SNS投稿のみ生成する場合の出力フォーマット。 |

### 編集時の注意点

1. **クォーテーション**: 各テキストは `"""`（トリプルクォート）で囲まれています。これを消さないように注意してください。
2. **JSON形式**: `OUTPUT_FORMAT` の部分はAIがJSON形式で出力するためのテンプレートです。キー名（`"title":` など）を変更するとプログラムが動かなくなる可能性があるため、値の説明部分のみ変更してください。
3. **波括弧 `{}`**: Pythonのf-string（フォーマット済み文字列）として扱われる箇所があるため、JSONの波括弧は `{{ }}` のように二重にする必要があります。

---

## 変更の反映（デプロイ）

ファイルを編集・保存しただけでは、本番環境（Cloud Functions）には反映されません。
変更を適用するには、以下のコマンドを実行してデプロイを行ってください。

```bash
./deploy.sh
```

デプロイには数分かかります。「Deploy complete!」と表示されれば完了です。
